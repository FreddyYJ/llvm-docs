Detailed Description

template<typename State, typename Result = void>
class clang::dataflow::MatchSwitchBuilder< State, Result >

Collects cases of a "match switch": a collection of matchers paired with callbacks, which together define a switch that can be applied to a Stmt.

This structure can simplify the definition of transfer functions that rely on pattern-matching.

For example, consider an analysis that handles particular function calls. It can define the MatchSwitch once, in the constructor of the analysis, and then reuse it each time that transfer is called, with a fresh state value.

MatchSwitch<TransferState<MyLattice> BuildSwitch() {
return MatchSwitchBuilder<TransferState<MyLattice>>()
.CaseOf(callExpr(callee(functionDecl(hasName("foo")))), TransferFooCall)
.CaseOf(callExpr(argumentCountIs(2),
callee(functionDecl(hasName("bar")))),
TransferBarCall)
.Build();
}
